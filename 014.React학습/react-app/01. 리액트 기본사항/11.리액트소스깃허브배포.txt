[ 리액트 배포용소스 깃허브에 호스팅하기 ]

<-- 5번 먼저 하고 1번 빌드 진행 -->

1. 리액트 SPA 개발환경 소스를 빌드한다
-> 빌드 명령어 : npm run build
-> 빌드 후 build 폴더가 생성됨(다시 빌드시 매번 갱신)
(참고: build폴더/node_modules폴더는 기본적으로 깃허브 소스관리폴더에서 제외->업로드하지않음)

2. 깃허브 실서버용 레파지토리를 생성한다

3. 깃허브 Pages에 서버를 공개한다
(단, 소스를 올린후 index 파일이 있어야 함!)

4. 깃허브 실서버용 소스 공간에 업로드 한다

5. Pages에 공개한 주소를 카피한다
(1) 깃허브 첫페이지에 링크 주소 등록(편리함)
(2) ! 개발소스 package.json파일 ["homepage":"깃허브 주소"] 를 등록해 준다 !
    (배포시에만 넣고 개발시엔 뺀다! - 있는 json을 백업)

    -깃허브 주소는... 
    https://아이디.github.io/레파지토리명
    ->>> 주의! 본 셋팅을 반드시 빌드전에 넣고 빌드한것!!

[ 배포 전 라우터 소스 수정하여 빌드하기 ]
- 깃허브는 기본적으로 SPA가 자동으로 적용되지 않음
- 라우터를 사용하여 개발한 경우 최상위 라우터 요소에 라우터 컴포넌트 요소의
    속성으로 아래와 같이 셋팅해야만 나온다(안하면 흰화면만 나옴)

라우터를 셋팅한 index.js파일에서... 
<BrowserRouter basename={process.env.PUBLIC_URL}>
 basename={process.env.PUBLIC_URL} 이 속성은 package.json파일의 "homepage"속성값을 읽어온다
 라우팅 연결시 이 주소로 정확한 라우팅이 되도록 셋팅해준다
 ->>> 주의! 개발소스로 띄울때는 주석처리하고 아무것도 없는  BrowserRouter를 사용한다!
